# Requirements Document

## Introduction

개인 블로그 플랫폼을 개발합니다. Java Spring Boot 백엔드 API와 React Next.js 프론트엔드로 구성된 풀스택 웹 애플리케이션입니다. 사용자가 웹 인터페이스를 통해 블로그 포스트를 작성, 편집, 관리할 수 있으며, 방문자들이 포스트를 읽고 댓글을 달 수 있는 완전한 블로그 시스템을 제공합니다.

## Requirements

### Requirement 1 - 사용자 인증 및 관리

**User Story:** 블로그 관리자로서, 안전한 로그인을 통해 블로그를 관리하고 싶습니다. 그래야 권한이 있는 사용자만 콘텐츠를 작성하고 수정할 수 있습니다.

#### Acceptance Criteria

1. WHEN 사용자가 로그인 페이지에서 올바른 이메일과 비밀번호를 입력하면 THEN 시스템은 JWT 토큰을 발급하고 대시보드로 리다이렉트해야 합니다
2. WHEN 사용자가 잘못된 인증 정보를 입력하면 THEN 시스템은 명확한 오류 메시지를 표시해야 합니다
3. WHEN 로그인된 사용자가 30분간 비활성 상태이면 THEN 시스템은 자동으로 로그아웃해야 합니다
4. WHEN 사용자가 로그아웃하면 THEN 시스템은 토큰을 무효화하고 로그인 페이지로 리다이렉트해야 합니다

### Requirement 2 - 포스트 작성 및 편집

**User Story:** 블로그 작성자로서, 웹 에디터를 통해 마크다운으로 포스트를 작성하고 편집하고 싶습니다. 그래야 언제 어디서나 쉽게 글을 쓸 수 있습니다.

#### Acceptance Criteria

1. WHEN 인증된 사용자가 새 포스트 작성 버튼을 클릭하면 THEN 시스템은 마크다운 에디터 페이지를 표시해야 합니다
2. WHEN 사용자가 제목, 내용, 태그를 입력하고 저장하면 THEN 시스템은 포스트를 데이터베이스에 저장해야 합니다
3. WHEN 사용자가 기존 포스트를 편집하면 THEN 시스템은 변경사항을 저장하고 수정 날짜를 업데이트해야 합니다
4. WHEN 사용자가 포스트를 임시저장하면 THEN 시스템은 초안 상태로 저장해야 합니다
5. WHEN 사용자가 마크다운을 작성하면 THEN 시스템은 실시간으로 미리보기를 제공해야 합니다

### Requirement 3 - 포스트 목록 및 상세 조회

**User Story:** 블로그 방문자로서, 포스트 목록을 보고 개별 포스트를 읽고 싶습니다. 그래야 관심 있는 콘텐츠를 쉽게 찾아 읽을 수 있습니다.

#### Acceptance Criteria

1. WHEN 사용자가 블로그 홈페이지에 접속하면 THEN 시스템은 최신 포스트 목록을 페이지네이션과 함께 표시해야 합니다
2. WHEN 사용자가 포스트 제목을 클릭하면 THEN 시스템은 해당 포스트의 전체 내용을 마크다운 렌더링하여 표시해야 합니다
3. WHEN 포스트가 10개 이상이면 THEN 시스템은 페이지네이션을 제공해야 합니다
4. WHEN 사용자가 모바일 기기로 접속하면 THEN 시스템은 반응형 레이아웃을 제공해야 합니다

### Requirement 4 - 댓글 시스템 (Facebook Comments로 변경됨)

**User Story:** 블로그 방문자로서, 포스트에 댓글을 달고 다른 사람의 댓글을 읽고 싶습니다. 그래야 작성자 및 다른 독자들과 소통할 수 있습니다.

#### Acceptance Criteria

1. WHEN 사용자가 포스트를 조회하면 THEN 시스템은 Facebook Comments 위젯을 표시해야 합니다
2. WHEN 사용자가 Facebook으로 로그인하면 THEN 댓글을 작성할 수 있어야 합니다
3. WHEN 댓글이 작성되면 THEN 시스템은 Facebook 서버에 저장하고 실시간으로 표시해야 합니다
4. WHEN 관리자가 댓글을 관리하려면 THEN Facebook Moderation Tool을 사용해야 합니다

### Requirement 5 - 태그 및 카테고리 시스템

**User Story:** 블로그 방문자로서, 태그나 카테고리로 포스트를 필터링하고 싶습니다. 그래야 관심 있는 주제의 글을 쉽게 찾을 수 있습니다.

#### Acceptance Criteria

1. WHEN 사용자가 태그를 클릭하면 THEN 시스템은 해당 태그가 포함된 포스트만 표시해야 합니다
2. WHEN 사용자가 카테고리를 선택하면 THEN 시스템은 해당 카테고리의 모든 포스트를 표시해야 합니다
3. WHEN 포스트에 여러 태그가 있으면 THEN 시스템은 모든 태그를 클릭 가능한 링크로 표시해야 합니다
4. WHEN 사용자가 태그 목록 페이지에 접속하면 THEN 시스템은 모든 태그와 각 태그의 포스트 수를 표시해야 합니다

### Requirement 6 - 검색 기능

**User Story:** 블로그 방문자로서, 키워드로 포스트를 검색하고 싶습니다. 그래야 원하는 정보를 빠르게 찾을 수 있습니다.

#### Acceptance Criteria

1. WHEN 사용자가 검색창에 키워드를 입력하고 검색하면 THEN 시스템은 제목과 내용에서 해당 키워드가 포함된 포스트를 표시해야 합니다
2. WHEN 검색 결과가 없으면 THEN 시스템은 "검색 결과가 없습니다" 메시지를 표시해야 합니다
3. WHEN 검색 결과가 많으면 THEN 시스템은 관련도 순으로 정렬하여 표시해야 합니다

### Requirement 7 - 관리자 대시보드

**User Story:** 블로그 관리자로서, 대시보드에서 포스트를 관리하고 싶습니다. 그래야 블로그 운영을 효율적으로 할 수 있습니다.

#### Acceptance Criteria

1. WHEN 관리자가 대시보드에 접속하면 THEN 시스템은 총 포스트 수, 조회수 등의 통계를 표시해야 합니다
2. WHEN 관리자가 포스트 관리 페이지에 접속하면 THEN 시스템은 모든 포스트의 목록과 편집/삭제 버튼을 제공해야 합니다
3. WHEN 관리자가 포스트를 삭제하면 THEN 시스템은 확인 메시지를 표시해야 합니다
4. WHEN 관리자가 통계 카드를 클릭하면 THEN 해당 관리 페이지로 이동해야 합니다

### Requirement 8 - 이미지 업로드 및 관리

**User Story:** 블로그 작성자로서, 포스트에 이미지를 업로드하고 관리하고 싶습니다. 그래야 시각적으로 풍부한 콘텐츠를 작성할 수 있습니다.

#### Acceptance Criteria

1. WHEN 사용자가 포스트 에디터에서 이미지를 업로드하면 THEN 시스템은 이미지를 서버에 저장하고 마크다운 링크를 생성해야 합니다
2. WHEN 이미지 파일 크기가 5MB를 초과하면 THEN 시스템은 오류 메시지를 표시해야 합니다
3. WHEN 지원하지 않는 파일 형식을 업로드하면 THEN 시스템은 "지원하지 않는 파일 형식입니다" 오류를 표시해야 합니다
4. WHEN 관리자가 이미지 관리 페이지에 접속하면 THEN 시스템은 업로드된 모든 이미지와 삭제 기능을 제공해야 합니다

### Requirement 9 - API 구현 (Next.js API Routes로 변경됨)

**User Story:** 개발자로서, API가 원활하게 동작하기를 원합니다. 그래야 사용자에게 끊김 없는 경험을 제공할 수 있습니다.

#### Acceptance Criteria

1. WHEN 프론트엔드에서 API를 호출하면 THEN Next.js API Routes가 JSON 응답을 제공해야 합니다
2. WHEN API 호출이 실패하면 THEN 프론트엔드는 적절한 오류 메시지를 사용자에게 표시해야 합니다
3. WHEN 인증이 필요한 API를 호출하면 THEN 시스템은 JWT 토큰을 검증해야 합니다
4. WHEN 파일 업로드 시 THEN 시스템은 public/uploads 디렉토리에 저장해야 합니다

### Requirement 10 - 성능 및 사용자 경험

**User Story:** 블로그 방문자로서, 빠르고 부드러운 사용자 경험을 원합니다. 그래야 편안하게 블로그를 이용할 수 있습니다.

#### Acceptance Criteria

1. WHEN 페이지가 로드되면 THEN 시스템은 3초 이내에 콘텐츠를 표시해야 합니다
2. WHEN 사용자가 페이지 간 이동하면 THEN 시스템은 로딩 인디케이터를 표시해야 합니다
3. WHEN 폼을 제출하면 THEN 시스템은 사용자에게 성공/실패 피드백을 제공해야 합니다
4. WHEN 네트워크 오류가 발생하면 THEN 시스템은 재시도 옵션을 제공해야 합니다

### Requirement 11 - 조회수 시스템 ✅

**User Story:** 블로그 운영자로서, 각 포스트의 인기도를 파악하고 싶습니다. 그래야 어떤 콘텐츠가 인기 있는지 알 수 있습니다.

#### Acceptance Criteria

1. WHEN 사용자가 포스트를 조회하면 THEN 시스템은 조회수를 1 증가시켜야 합니다
2. WHEN 같은 IP 주소에서 동일한 포스트를 여러 번 조회하면 THEN 시스템은 조회수를 중복으로 증가시키지 않아야 합니다
3. WHEN 포스트 목록을 표시하면 THEN 시스템은 각 포스트의 조회수를 표시해야 합니다
4. WHEN 포스트 상세 페이지를 표시하면 THEN 시스템은 조회수를 표시해야 합니다
5. WHEN 관리자가 대시보드에 접속하면 THEN 시스템은 전체 조회수 통계를 표시해야 합니다

### Requirement 12 - UI/UX 개선 ✅

**User Story:** 블로그 방문자로서, 세련되고 읽기 편한 인터페이스를 원합니다. 그래야 콘텐츠에 집중할 수 있습니다.

#### Acceptance Criteria

1. WHEN 사용자가 포스트를 읽으면 THEN Medium/Notion 스타일의 최적화된 타이포그래피가 적용되어야 합니다
2. WHEN 사용자가 카드에 마우스를 올리면 THEN 부드러운 애니메이션 효과가 나타나야 합니다
3. WHEN 사용자가 버튼을 클릭하면 THEN 아이콘과 함께 명확한 액션이 표시되어야 합니다
4. WHEN 페이지를 스크롤하면 THEN 헤더가 상단에 고정되어야 합니다
